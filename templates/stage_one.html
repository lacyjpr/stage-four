Stage 1.1 The Basics of the Web and HTML

The World Wide Web

The WWW is a collection of html documents. HTML is the basis for most every web page. Hyperlinks link the web together. Major Pieces of the web: You, Your computer and browser, The Internet, Servers, HTTP. HTTP (Hyper text transfer protocol) moves HTML files from servers to your browser.

HTML

Hypertext Markup Language. Made up of: Text, content, markup, references to other documents, links to other pages.

HTML Markup

Plain text is rendered as plain text in HTML. For something different, we need markup:

<name> contents </name>

<b>bold</b>

<em>emphasis (italics) </em>

Computers Are Stupid

Computers only reflect the given input. They can only interpret things literally. Typos and other mistakes won't be corrected by a computer, magnifying errors.

HTML Attributes

<TAG ATTR="value"> contents </tag>

anchor tag (for making links):

<a href="www.reddit.com"> derp </a>

image tag (for including images)

<img src="url" alt="text">

Whitespace

In HTML, all text and white space is rendered on a single line until formatted with tags.

<br> line break </br>( void)[inline]

<p> paragraph </p>[ block]

<span> text </span> inline

<div> text </div> block

HTML Documents

HTML documents are structured like this:

<!DOCTYPE html>
<html>
<head>
	<title>Title</title>
</head>
<body>
	content
</body>
</html>

Stage 1.2 Creating a Structured Document

HTML, CSS & Javascript

Think of a website like your house: HTML files are like the structure, CSS files are the style ( decorating.) Javascript files are the interactive components ( garage door opener.) Example: Wikipedia--to see the inner structure use Chrome: Hamburger Menu, More Tools, Developer Tools. Shows you the structure of a web page.

Tree Like Structure

HTML uses a tree like structure, sort of like a family tree, a branching hierarchical structure.

HTML-CSS-DOM

How the browser knows what style to apply & where that style comes from:

HTML: Language of the web (Tags) Defines the structure & content of the page.

CSS: Defines the style. Changes how elements on the page look.

DOM: Document Object Model -- Tree like structure. Tags make up the elements of the tree.

Boxes

Everything on a web page is a box. This makes things easy to rearrange. To boxify a design, cut up the boxes physically or conceptually. Work from big to small . Boxes are defined with div tags. (And <p> tags and many new semantic ones)

To mark div tags to apply style later, give each element a class attribute. (Like labels on boxes for movers.) Use class names that make sense, so you know the purpose by glancing at the code: See www.udacity.com/course/viewer#!/c-nd000/l-4137489485/m-2635788572 

<div class="title">My App</div>
<div class="app">
	<div class="screenshot">image</div>
	<div class="description">text</div>
</div>

Stage 1.3 Adding CSS Style and HTML Structure

Avoid Repetition

Its essential in all areas of programming. It saves time and effort, leading to efficient code and reducing errors. CSS allows programmers to avoid repetition, improving their code.

Tool Tips

We'll be downloading Sublime Text. As you learn each new tool during this Nanodegree, you'll need to spend time with each one learning now to set it up and use it.

How CSS Allows Programmers To Avoid Repetition

CSS Stands for "Cascading Style Sheets," with "Cascading" being the most important concept. You can have several CSS files referenced in HTML. All of them will be used to determine the final style for the page. Analogy: Laws-- National, State laws, Municipal laws, house rules. CSS is different . With CSS the most specific rule is applied to every element. If a house rule addresses the same topic as a National law, then the house rule would be what counts. Cascading means "most specific rule applied."

Inheritance

According to wikipedia https://en.wikipedia.org/wiki/Cascading_Style_Sheets#Inheritance "Inheritance is the mechanism by which properties are applied not only to a specified element, but also to its descendants.[11] Inheritance relies on the document tree, which is the hierarchy of (X)HTML elements in a page based on nesting. Descendant elements may inherit CSS property values from any ancestor element enclosing them."

Selectors

In CSS, you first write a selector. This defines what elements of a page a particular style will apply to. You can select an html tag and that rule will apply to all tags on the page. Or, you can select elements by class:

.class {

attribute:value

}

.description {

color: red;

}

CSS Attribute reference: https://developer.mozilla.org/en-US/docs/Web/CSS/Reference

The Box Model

The box is made up of Margin, Border, Padding and Content.

Content: Image or text on the website. Think of it as the nucleus

Padding: Clears an area around the content.

Border: Goes around padding and content. Color inherited from the color property of the box.

Margin: Clears an area around the border. Doesn't have a background color, completely transparent. Think of as space between boxes.

Sizing Boxes

The size of the element is equal to the sum of the border padding and content widths. This makes recalculating element size difficult.

Easier method: CSS style

* {
	-webkit-box-sizing: border-box;
	-moz-box-sizing: border-box;
	-ms-box-sizing: border-box;
	box-sizing: border-box;
}

This makes the calculation of the element size include both the border and padding, making it easier to do layouts.

Setting the size of a box: Usually in pixels, also possible to do as a percentage. That way the element resizes with the screen size.

Positioning Boxes

One of many techniques: Flex box layout. If you want display elements to be next to each other, you can change the display attribute of the parent container to value flex.

For example: <div class="app" style="display: flex;" >

Code, Test & Refine

Write code, test & see how it looks vs the design mock up, make refinements and modifications so it looks closer to the design. Repeat this process until the page looks like the design. How do you know where to start testing? How do you know what to refine? First, get the structure of the page right by boxifying it. Then move on to the tags: If something needs to be a heading, make it one. Then, think about the size of the boxes. Then, how to position them where you want them. So:

  1. Look for natural boxes.

  2. Look for repeated styles & semantic elements.

  3. Write your html.

  4. Apply styles (from biggest to smallest.)

  5. Fix things. Developer tools in your web browser can also be a big help without messing with the html editor.

Refining the Design: As you refine the design answer the following questions:

  1. What margin size gets the design closer to the mock?

  2. What is the background color for the header?

Validation

Its very important that the html and css you write is correct and complies with standards. If incorrect, it might display right in one browser but not another.

HTML verifier: http://validator.w3.org/#validate_by_input

CSS verifier: http://jigsaw.w3.org/css-validator/#validate_by_input
