Stage 2.1 Introduction to Serious Programming

What Computer Science is About

Computer Science is about how to solve problems by breaking them into smaller pieces and then precisely and mechanically describing a series of steps that you can use to solve each piece, and those steps can be executed by a computer.

Programming

The core of computer science. Most machines are designed to do one thing, like a toaster. Without a program, a computer is even less useful than a toaster. The program tells the computer what to do. The power of a computer, unlike a toaster, is that a computer can do anything. Anything we can figure out how to write a program for, we can make the computer do. A program needs to be a very precise sequence of steps . The computer by itself doesn't know how to do anything, it has a few simple instructions that it can execute. To make a program do something useful, we need to put those instructions together in a way that does what we want. The power of the computer is that it can execute those steps super fast. The program gives us a way to tell the computer what steps to take. There are many different languages for programming computers. The language we will be learning is called Python. Python gives us a high level language to write programs. That means instead of running directly on the computer, the programs we write will be input into the Python program, which runs on the computer. Python is called an interpreter, that means it runs our programs, it interprets them, it executes the program that we wrote in the Python language.

Why We Need Programming Languages

Why do we need to invent new languages like Python to program computers, rather that using natural languages like English or Mandarin? 1. Ambiguity: Natural languages are inherently ambiguous. Different people interpret the same sentence different ways. When we program computers we need to make sure the computer interprets the language the exact same way the programmer does. Example: Biweekly can mean twice a week or once every two weeks according to the dictionary. 2.Verbosity. Natural languages are very verbose. To write a program we need to describe exactly what the computer should do in a very precise sequence of steps. If we had to describe all those details using natural language, it would require a huge amount of text. With a language like Python we only need a few lines of code to describe a complex thing in a step by step way.

Grammar

Like English, Python has a grammar that defines what strings are in the language. English rule: You can make a sentence by combining a subject with a verb & object . Backus-Naur Form is a way of writing grammars. The purpose is to describe languages in a way that is simple and concise. On the left there is a [Non-terminal] (something we aren't finished with) then an arrow, and on the right side, a replacement. The replacement can be anything, it can be a sequence of non-terminals, it can be one non-terminal, it can also be a terminal. Once we get to a terminal, there is nothing else we can replace it with. We can form a sentence by starting from some non- terminal, then by following the rules, we keep replacing non- terminals with their replacements until we're left with only terminals. Derivation: Starting from non-terminals follow the rules to come up with a sequence of terminals. Important thing about replacement grammars is that we can describe an infinitely large language with a small set of rules.

Python Grammar

Python grammar is much stricter than English. Ungrammatical speech can still be understood in English. In Python, the code must match the language grammar exactly. print 2 + gives a syntax error.

Python Grammar for Arithmetic Expressions

Expression->Expression Operator Expression

Expression->Number

Operator->+

Operator->*

Number->0,1,...

Expression->(Expression)

Example Derivation: Expression->Expression Operator Expression, Number->1, Operator->+, Number->1, 1+1.

Stage 2.2 Variables and Strings

Variables

Our answer to the last quiz ( print 299792458 * 100 * 1.0 / 1000000000) would have been a lot easier to read and a lot more useful if we used names to keep track of values, instead of writing out those big numbers, especially numbers as big as the speed of light. Python provides a way to do it, its called the variable. We can use a variable to create a name and use that name to refer to a variable. Way to introduce a variable is using an Assignment Statement: Name = Expression. After the Assignment Statement, the name refers to the value that the expression has. The name can be any sequence of letters and numbers as well as underscores, as long as it starts with a letter or underscore. Example: speed_of_light = 299792458.

When we print speed_of_light, we'll see 299792458. We can use it in expressions too: If we want to convert it to centimeters instead of meters we can multiply by 100:

print speed_of_light * 100 gives the result 29979245800.

Another variable: billionth = 1.0 / 1000000000. Another: nanostick = speed_of_light * billionth * 100 (Nanostick is the length light travels in a nanosecond.) print nanostick gives the result 29.9792458

Variables are very useful, they make our programs a lot easier to understand. They also mean that we can use the same expression, changing the values of the variables to compute different things.

Comments in Python: # from the start of the to the end of the line is a comment.

Variables Can Vary

Once we define a variable, we can change the value. When we use the variable again, it refers to the new value.

Example:

days = 7 * 7 (days now refers to the value 49)

days = 48 (days now refers to the value 48)

We can use variables in their own assignment statements:

days = days - 1 (days now refers to the value 47 )

We can do the same thing again: days = days -1 ( days now refers to the value 46)

= This is not an equals symbol. = means assignment in Python. Think of it as an arrow that says put whatever value the right side evaluates to into the name on the left side.

Strings

In the early days of computing, people thought of computers as super powerful calculators for doing computations like simulating nuclear weapons, computing ballistic tables, or breaking encryption. There is no reason to limit computers to numbers: They can operate on any kind of data we want and it gets much more interesting when we operate on data besides just numbers.

A string is just a sequence of characters surrounded by quotes. Example of a string in Python: 'I am a string!' If we want we can use double quotes instead: "I prefer double quotes!" The only requirement is that if we begin with a single quote we end with one, the same with double quotes. "I'm happy I started with a double quote!"

print 'Hello'

print "Hello"

Both return the same thing.

Assigning a string to a variable is just like assigning a number to a variable: name = "John"

print "Hello " + name

The + operator on strings means something different than with numbers. With strings it means concatenation

string + string ->concatenation of the two strings. [You need to add spacing as necessary to the strings to format output.] You cannot use the + operator with a string and a number [cannot concatenate 'str' and 'int' objects.] You can multiply strings: print '!' * 12 results in 12 exclamation points.

Indexing Strings

One of the things we can do with strings that we can't do with numbers is extract subsequences from the strings.

If we have a string, we can use the square brackets to extract parts of that string: <string> [<expression>]

'udacity'[0] would extract the 0 character from the string. Characters in a string are indexed starting from zero. The result of 'udacity'[0] is the string 'u.'

The expression inside the square bracket can be anything that evaluates to a number. 'udacity'[1+1] returns 'a .'

name='Dave'

name[0] returns 'D'

You can use negative numbers, in which case it starts counting from the back of the string.

Selecting Subsequences From Strings

<string>[<expression >:<expression>]

The value of the first expression we'll call start, the value of the second expression we'll call stop.

The result is a string that is a subsequence of the characters in the string starting from position start and ending with position stop but not including that character. So start to stop -1.

word = 'assume'

print word[4:6]

returns me

print word[4:]

selects from position 4 to the end.

print word[:2]

selects from the beginning

print word [:]

selects the entire string

Finding Strings in Strings

The find operation gives us a way in a big string to find some substring that we're looking for. The way we use find is a little different because find is a method (a built in procedure provided by Python.)

Find is a procedure that operates on strings <string>.find(<string>) The second string is the one we want to find in the first.

The output of find is the position in the string where that substring is found (The first occurrence.) A number that gives the first position in the search string where the target string appears . If the target string is not found, output will be -1 .

Finding with Numbers

Find returns the first position, but there might be other occurrences we want to find.

The way to find them is to pass a second parameter:

<string>.find(<string>, <number>)

What find will output is the number of the position in the search string where the target string first appears at or after the <number.>

Functions [or Procedures]

How to use a procedure [function]:

<procedure>(<input>, <input>,...)

There can be any number of inputs, but they have to match the number of inputs the procedure expects. Inputs are sometimes called operands or arguments.

We've already done something similar to this. In 2.2 we learned about using find on strings:

find('au',3)

With find, you would pass in one or 2 inputs. The first input was a string (that you were looking for.) The second input could be a number (the position where you start looking for that string.) We used find in many ways in stage 2.2.

Find is a little different from the procedures that you define yourself. First of all, its built in. Secondly, instead of just having find, we had another input in front of the function, the string we were doing the find in, say its the variable page:

page.find('au',3)

That is really another input to find. We'll talk later about why its done differently. But its very similar to calling a procedure where one of the inputs is on the left and the other two are on the right.

For all the procedures you define yourself, we won't have any object to invoke them on like the string in find. We'll just have the procedure to call and the arguments or operands or inputs to pass in.

How this works with a simple procedure:

def rest_of_string(s):

return s[1:]

This will evaluate to the string with the first letter removed.

How to use this procedure:

print rest_of_string('audacity')

What happens when we call a procedure like this: Execution will jump into the body of the procedure. What the interpreter is doing is jumping to run the code inside the procedure. It will assign to the parameters the values passed in as the inputs. Think of it as being an assignment that says now the value of s = 'audacity' . Now we're going to evaluate the body of the procedure. In this case, there's only one statement, the return statement. We're going to find the value of s[1:]. The result of that is the string 'udacity'. Then we get to the return. What return means is that we're going to jump back to where we called the procedure, but now we actually have a result. So, when we jump back, the value that rest_of_string('audacity') evaluates to is whatever value we returned. In this case, its the string 'udacity'

Anything we're going to do in the rest of the course, and anything almost anyone does in programming is all about defining procedures and using procedures. We can think of procedures in terms of mapping inputs to outputs.

Example:

	c = a + b
	return c

Using a function:

Write the name of the function then the parameters we want to input.

Functions Avoid Repetition

Functions avoid repetition because they are infinitely reusable. Once you write a function you can use it many times in your code without having to define it again.

Functions Without a Return

What does this function do?

def sum(a,b):
	a = a + b

It really does nothing. It doesn't have a return, so it doesn't produce any output. It doesn't actually modify the value of the first input in any way thats visible to the caller. It does modify the value of a, inside the function, but the caller can't actually see it.

The result of the procedure in Python is None. Python uses None to mean "there's no value."

Lets add a return statement:

def sum(a,b):
	a = a + b
	return a
print sum(2, 123)

Now when we run this, we get the result 125.

Functions and Variables

If we pass in variables (without the return) it still does not change their value.

def sum(a,b):
	a=a+b
a=2
b=123
print sum(a,b)
print a

Does the value of a change after running this? No. The reason is what gets passed in is the value, not the name of the variable. What gets passed in is the value of a, in this case 2. We don't actually modify what the variable is.

More explanation:

a=2

b=123

Name a refers to the value 2. Name b refers to the value 123.

Then, we have our procedure:

def sum(a,b):
 a = a + b

When we call sum, sum(a,b) we have the names of our parameters. We're going to have a name inside sum called a. That a refers to whatever value is passed in when sum is called. a refers to the value 2, b refers to the value 123. When we run sum, we do the assignment, that is going to produce the new number, adding 2 + 123, and we get the value 125. Then we do the assignment. The assignment changes what a refers to. Now instead of referring to the number 2, the a that corresponds to the parameter to the procedure sum, now refers to 125.

Note that the original value of a, the one we named outside the function, still refers to 2. When we come back from the procedure, all the parameters that were visible inside the body of sum are no longer visible. They were there for that execution, but they're done, we can't use those names any more . They were only visible during that execution of the procedure.

Stage 2.4 Control Flow & Loops: If and While

Making Decisions

Everything we've had to do so far has been pretty limited. We had to do the same thing on all data. We couldn't do anything that really depended on what the data was.

What we're going to do next is to figure out a way to make code behave differently based on decisions. The first thing we're going to do is figure out some ways to make comparisons so we have a way to test and decide what we should do.

Python provides lots of different operators for doing comparisons. They're things similar to what we've used in math: < we have a less than sign that compares two numbers. > we have a greater than. <= we have a less than or equal to. All of these operate on numbers. We can have a <number> followed by a comparison operator > <operator> followed by another number <number >. This is very similar to the grammar we saw earlier for arithmetic expressions, but now instead of having a plus or times we have something that does a comparison. The output of a comparison is not a number, its a boolean value. A boolean value is one of two things: Its either the value True or the value False. Note that in Python, the boolean values True and False are capitalized.

Examples:

print 2 < 3

The result is True

print 21 < 3

The result is False

We can have any expression we want with the comparison:

print 7 * 3 < 21

The result is False, because 7*3 is equal to 21

Another comparison operator we can use is "not equal to."

print 7 * 3 != 21

The result is False, because 7*3 is equal to 21.

If we want to do equality comparison, we don't use the equals sign, we use two equals signs. We call that the "double equal."

print 7 * 3 == 21

The result is True.

Why is the equality comparison done using == instead of =? Because = means assignment.

Example:

i = 21 assigns the value 21 to the variable i . i == 21 is a comparison, it evaluates to either True or False depending on whether the value of i is 21.

If Statements

If:

Now we know some ways to make comparisons, we want to use them to make decisions, to make our code do something different depending on the result of a comparison. The way to do that is to use an if statement.

The structure of an if statement is we have the keyword if followed by a comparison, we'll call that the <Test Expression>, followed by a colon.

if <Test Expression>:

<Block>

Then, inside the if, we have the block: The block is the code that will run if the test expression is True. If the test expression doesn't evaluate to True , then the block doesn't execute. As with function definitions, we know the end of the if because of the indentation. All the statements inside the block are executed only when the test expression is True. The next statement, that is not indented, is going to be executed whether or not the test expression is True.

Example of a function using an if statement:

def absolute(x):
	if x< 0:
		x = -x
	return x

What this does is give us the absolute value of the number thats passed in.

Else:

Start with this:

def bigger(a,b):
	if a > b:
		return a
	return b

Instead of just having what we had before:

if <Test Expression>:

<Block>

What we want is something that executes when the test expression is False.

We can do that by using an else.

if <Test Expression>:

<Block>

else:

<Block>

As before, the indentation tells us when we're done with the block.

How to rewrite bigger using else:

def bigger(a,b):
	if a > b:
		return a
	else:
		return b

This means the same thing as the first function, and in this case neither is particularly better. but there are lots of cases where it makes things work a lot better by using else.

One way using else makes things better:

def bigger(a,b):
	if a > b:
		r = a
	else:
		r = b

Or:

<Expression> or <Expression>

If the first expression evaluates to True, the value is True and the second expression is not evaluated. If the first expression evaluates to False , the value is the value of the second expression.

Example:

def is_friend(name):
	return name[0] == 'D' or name[0] == 'N'
print is_friend('Doug')
print is_friend('Nicole')
print is_friend('Fred')

or gives us the logical or of the two operands on its left and right.

Examples:

print True or False

The value is True

print False or True

also has the value True

print True or True

is also True

print False or False

is False

The one important thing that or does that is different from other operators is that it only evaluates what it needs to.

Example:

print True or this_is_an_error

evaluates to True

this_is_an_error is an undefined variable, its an error. But, when we evaluate the or expression, because we already found that the first part was True , we don't get an error.

print False or this_is_an_error

In this case, its necessary to evaluate the second operand ( since the first one is False,) and we'll get an error.

Loops

Loops are a way to do things over and over again.

The kind of loop we'll introduce first is called a while loop.

Syntax is this:

while <Test Expression>:

<Block>

The block is a sequence of instructions. With if, the block is executed 0 or 1 times. With while, the block can be executed any number of times. It keeps going as long as the test expression is True. Once the test expression is False, we go to the next expression. The loop could keep going forever: There is no requirement that the test expression eventually becomes False. (We call this an infinite loop.)

Example:

i=0
while i < 10:
	print i
	i = i + 1

This example is not an infinite loop because we include a counter, i that makes the test expression False when it gets to 10.

Break Statements

break gives us a way to stop a loop even while the test condition is True.

while <Test Expression>:

<Code>

if <Break Test>:

break #once break happens, we jump to the code after the while

<More Code>

Example:

Start with this:

def print_numbers(n):
	i=1
	while i <= n:
		print i
		i = i + 1

Instead of that, use a break:

def print_numbers(n):
	i = 1
	while True:
		if i > n:
			break
		print i
		i = i + 1

Stage 2.5 Debugging

Bugs Happen!

Sometimes when you try something in code, it just doesn't work the first time. These errors are called bugs.

Writers spend a lot of time proof reading and scientists spend a lot of time checking their results. But it was a surprise to early programmers that debugging was going to be a thing. A good bit of time will be spent finding errors in programs. Andy thought that every bug he introduced was some failure on his part as a programmer. He realizes now that bugs are inevitable. The important thing isn't to avoid them entirely, its to develop a strategy for systematically getting rid of them. Its not so much about spotting typos or logic errors as it is about building a correct mental model of how that system works so that you can find out what isn't working.

Here are some techniques for making debugging part of your learning process:

Examine error messages

Sometimes its really obvious that code has a bug because the bug makes the code crash. When it crashes, it usually gives some sort of error message. Error messages can be confusing, but they do give some kind of information besides just "it didn't work, try something different. " When Python code crashes, the message it gives is called a Traceback. The traceback tells you what line it crashed on, what file it was running, and how it got there. The information the traceback gives you, especially the line number and function name, can focus your attention on the part of the code that might be going wrong. Even if the root of the problem isn't on that exact line, its somewhere upstream of that line. We can also look up the error message to find out what to do about it.

Work from a Working Example

When your learning to use a new feature or language, you'll often look at example code to give you a model of what to write. Then you'll write code that's based on the example code, but change it to do what you want it to. If that doesn't work, what then? A lot of new programmers try making random changes, when that doesn't work, they ask for help. Here's another approach: When your code doesn't work, set it aside, comment it out. Then copy and paste in the example code. Run it and make sure it does what its supposed to. Then, modify it in small steps until it does what you wanted your code to do. Once you're done, see if you can find where your original code had a bug . Make sure the examples work! When you're working from example code, you do have to exercise some care, or could inherit bugs from it.

Check Intermediate Results

Bugs that don't crash your program can be a lot trickier than ones that do. When a function calculates an incorrect value, finding out where it went wrong often involves splitting the function up into smaller pieces and looking at the value of variables in each of those pieces. One basic way to do this is to introduce print statements that output the variables where you can see them. Once you are done debugging, be sure to remove the print statements or your code will print debugging information when you don't want it to.

Keep and Compare Old Versions

If you have something almost, but not quite working, and you're getting frustrated with it, it can be tempting to throw it away and try again. But, sometimes you're really close on your first try and your second try isn't as close. Then you want to be able to get back to the first version. For short functions you can make this easier by commenting out the old version. In Python, just put a hash mark (#) before each line of the function. Or, you can turn it into a big string by putting triple quotes around it. Text editors do have an undo feature, but it doesn't usually let you go back between multiple different versions of your code. Its not really good enough to let you compare different versions of code. Undo is good for when you accidentally mess something up and immediately notice it. But you can't rely on undo alone to compare different versions of code, or to get back to an arbitrary version you previously had. You can copy code into a different file. Files aren't exactly expensive after all. Just make sure you save them where you can find them. Another reason to keep old versions is that when you do ask for help, you can share two or three different things you've tried. Software engineers use version control systems like Git, which are pretty much super-powerful undo systems that let you navigate to any version of the code, and have multiple modified versions and share different versions and so on. Keeping old versions isn't just for beginners, its how software engineering really works.

Debugging as a Scientific Process

Now you have a few more tools to approach problems in your code. This is just the beginning. When you really get into it, debugging isn't just correcting errors or even just figuring out how you made a mistake. Its a scientific process of examining code to discover things you didn't expect in it. Debugging is a really important skill to learn if you want to do software engineering. In real world computing systems, bugs cause problems that are a lot bigger than just getting a weird error message. Correctly locating errors is a big part of writing reliable software. In fact, we have a whole course on "A Scientific Approach to Debugging." After you finish your current projects you might want to take a look at that next.

Stage 2.6 Structured Data: Lists & For Loops

Structured Data

The closest thing we've seen to structured data is the string data type. A string is a kind of structured data, we can break it down into its characters. The string is a sequence of characters, and we can operate on subsequences of the string. What we can do with strings is somewhat limited because the only thing we can put in a string is a character.

Today, we're going to introduce the list data type. Lists are much more powerful than strings. Whereas for a string, all the elements had to be characters, in a list the elements can be anything we want. They could be characters, they could be strings, they could be numbers, they could also be other lists.

Example:

String

Sequence of characters

s = 'yabba!'

List

Sequence of anything
p = ['y','a','b','b','a','!']

With a list, instead of using quotes to identify the list, we use square brackets, and the elements are separated by commas. Just like with a string, we can assign the list to a variable.

With a string, we can use square brackets to select elements: s[0] when we index element 0, we'll get the first element of the string, in this case 'y'.

With lists, we can also use square brackets to access elements: p[0] evaluates to the first element of p , which is the string containing the single letter 'y' .

With strings, we could use the colon inside square brackets to select a substring of more than one character: s[2:4] gives us 'bb'.

We can do the same thing with lists: p[2:4] Instead of returning a string, it will return a list containing those elements: ['b','b']

The general grammar for constructing a list is a square bracket, followed by a list of any number of expressions, where the expressions are separated by commas:

<list> -> [<expression>, <expression>, . ..]

We could create a list using just two square brackets. This would create a list containing zero elements, also known as the empty list.

We could create a list containing one element, or we could create a list with many elements, as we did in the first example.

Nested Lists

All of the examples we've seen so far, all of the elements in the list have the same type. We had a list where all the elements were strings, we had a list where all the elements were numbers.

There is no restriction on lists, we can make all of the elements any type we want. We can mix and match different things in one list.

Example:

mixed_up = ['apple', 3, 'oranges', 27,

We've put some strings and some numbers together, thats perfectly okay.

mixed_up = ['apple', 3, 'oranges', 27, [1, 2,

We can have another list inside that:

mixed_up = ['apple', 3, 'oranges', 27, [1, 2, ['alpha', 'beta']]]

We've defined a new variable mixed_up. It contains a string, a number, then a list, the fifth element is itself a list that contains a number, a number and contains another list.

This example shows the versatility of lists, that we can put anything we want in them. It doesn't seem particularly useful because there is no structure to the things we've put in the list.

Mutation

There are two main differences between strings and lists. The first is that a list can hold anything we want but a string can only be made up of characters. The second big difference between list and strings is that lists support mutation. Mutation sounds a little bit scary and in some ways it is. It makes it much harder to understand what our programs mean, but its also very powerful.

What mutation means is that we can change the value of a list after we've created it.

To explain what that means and to see why its so different from the behavior we can get with strings, lets recap what happens with strings:

We can create a string and store it in a variable:

s = 'Hello'

We can change the value of the variable by creating a new string:

s = 'Yello'

Whats going on there is changing the value of s, not changing the value of the string. s = 'Yello' doesn't change the value of 'Hello', we just don't have a way to refer to that string any more.

With strings we can also use the + operator to concatenate two strings.

s = s + 'w'

It looks like this is changing the value of the string. You might think the result of this is the string that was 'Yello' is now changed to 'Yellow'. Thats not actually what happens. We don't actually modify the string we had. What happens when we use the + operator is it creates a completely new string, where the value of the new string is the result of concatenating the two strings. The assignment changes what the variable refers to, to refer to the new string.

A List of Strings

p = ['H', 'e', 'l', 'l', 'o']

We've created a list, using the square brackets, that contains five strings, each one is a one letter string spelling out the word Hello. We've stored that list in the variable p.

We could do the same thing we did with strings. We could reassign p to a new value:

p = ['Y', 'e', 'l', 'l' 'o']

Instead, we're going to mutate the list we've already created.

We can do that by modifying the value of its elements:

p[0] = 'Y'

Here we have an assignment where on the left side of the assignment we have p[0], this means select from the list that p refers to, find position 0, and instead of getting the value of that position, since its an assignment, we're going to replace that value with what's on the right side of the assignment, in this case the string containing the single letter 'Y'. 'H' will be replaced by 'Y'. Note that we did not create a new list, p still refers to the same list, but we've changed the value of that list.

Yello Mutation

Mutation modifies the existing object, but you can't really see the difference from what we've done with strings. Where we'll really see the difference is when we introduce a new variable.

Lets go back to the example we had before:

p = ['H', 'e', 'l', 'l', 'o']

p[0] = 'Y'

Now we're going to add an extra assignment statement and introduce a new variable:

q = p

We introduce the variable q, and we assign p to q. That means the value of p, the list, is what q will refer to. After the assignment p and q refer to the same list.

Suppose we did an assignment statement to modify one of the elements of q:

q[4] = '!'

That will change the 4th element of q. It also changes the value of p, even though the assignment statement didn't include p. The fact that p and q refer to the same object means that it changes the value of p.

A key difference between mutable objects and immutable objects is once an object is mutable, then we have to worry about other variables that might refer to the same object. We can change the value of that object, and it affects not just the variable that we think we changed, it affects the value of other variables as well.

Aliasing

When we have two different ways to refer to the same object, that is called aliasing.

Aliasing in programs is similar to the way we think about aliasing in spy craft. We can refer to James Bond by his name or his spy name, 007. Both of those are ways to refer to the same person. If something happens that changes the state of James Bond, it affects the state of both James Bond and 007. Both names refer to the same person, so whatever state that person has is true for both names.

Aliasing for variable names has the same property. If we have two variable names that refer to the same object, say p and q, any change we make to the object p refers to also affects the value that q refers to.

List Operations

Lists are very powerful, by using mutation, by having lists that can contain other lists or any other kind of data we want, we can build very complex data structures and we can do lots of interesting things. We've seen that we can use lists to store complex data, that elements of the list can be any type we want, including other lists, and we've seen we can use mutation to change the value of the list, and that mutation is visible to any reference to the same list object.

One more list operation will allow us to add a new element at the end of the list:

append

append is like a procedure, but its a method. We use it similar to the way we use find on strings:

<list>.append(<element>)

append will add a new element to the end of the list. The important thing about append is its mutating the list that its invoked on, its not creating a new list, its mutating the old list.

Example:

stooges=['Moe', 'Larry', 'Curly']

Lets assume that instead of replacing Curly we want to end up with four stooges. We'll add Shemp and keep the other three as they are.

We have not created a new list. There is no assignment from the result of append. What we've done is modify the value that stooges refers to by adding a new element to it.

+ (plus.)

<list> + <list>

+ behaves very similar to the concatenation operation for strings:

[0, 1] + [2, 3]

Result is the list [0, 1, 2, 3]

What + does, like concatenation for strings, is it produces a new list. It doesn't mutate either of the input lists.

One other list operator:

len

len is short for length.

len(<list>)

len is a function call, its just a function already included in Python. We pass into len the object we want to know the length of.

len works for many other things than lists. It also works for strings, it works for any object thats a collection of things.

The output from len is the number of elements in the input.

len([0,1]) result is 2

len(['a', ['b,' ['c']]]) result is also 2. It looks like there are more elements, but len is only counting the outer elements. If one of the elements of a list is a list, it doesn't matter how many elements it contains, it only contributes one to the length of the original list.

len ("Udacity") We can use len on a string. The output will be the number of characters in a string. In this case, 7.

index

<list>.index(<value>)

The index method is invoked on a list, you pass in a value, and the output is the first position that value exists in the list. When we invoke index on a list that doesn't contain value we pass in, index doesn't return -1, it gives us an error.

To help deal with that error, we have two more operators:

in

<value> in <list>

Syntax: a value, the in word, followed by the list.

Returns a boolean True or False

not in

<value> not in <list>

If the value is not in the list the result is True. If it is in the list the result is False.

Loops on Lists

How to loop through a list: Since lists are collections of things, its very useful to be able to go through a list and do something with every element of the list.

Remember the while loop from the previous unit:

while <test>:

<Block>

As a reminder, this is the way the while loop works: we have a while followed by a test expression, which is something that evaluates to True or False , when it evaluates to True, that means to execute the block, which can be of any number of statements we want. We keep executing the block as long as the test expression evaluates to True.

We can use a while loop on a list:

def print_all_elements(p):
i = 0
while i < len(p)
	print p[i]
	i = i + 1
For Loops

We can write all the code we need to loop through the elements of a list just using while. It would just be a little more complicated than we want.

Python provides a simpler way to loop through the elements of a list: The for loop.

for <name> in <list>:

<Block>

The structure of a for loop is like this: We have the keyword for, followed by a name, (this is a new name for a variable that we can introduce,) the the keyword in, followed by a list, followed by a colon, with a block inside the for.

What a for loop like this means is that for each element in the list, we're going to assign that element to the <name> and evaluate the block, and we'll do that in order, going through the list.

Using for we can define print_all_elements using far less elements than we did using while:

def print_all_elements(p:)
	for e in p:
		print e

How to Solve Problems

Solving problems is one of the most important skills you can learn. Calling it a skill is really an understatement. Nearly everything you can do can be viewed as solving problems. Improving as a problem solver is a lifelong challenge.

One short lesson is not going to be enough to make anyone an expert problem solver, but for this lesson we're going to look at how to approach more complex programming problems, and hopefully that will help you with solving any problem you are faced with from now on.

It doesn't really make any sense to talk about solving problems without talking about particular problems. We're going to look at a specific practice problem and talk about how to go about solving it. Our goal by looking at a particular problem is not just to solve that problem, but to draw general lessons.

Days Between Dates

Given your birthday and the current date, calculate your age in days. Compensate for leap days. Assume that the birthday and current date are correct dates (and no time travel.) Simply put, if you were born 1 Jan 2012 and today's date is 2 Jan 2012 you are 1 day old.

First Step

What is the first thing we should do to solve a problem like this one?

Make sure we understand the problem. Other steps may get us to a solution, but they are more likely to fail unless we understand the problem in the first place.

Understanding a Computational Problem

Given your birthday and the current date, calculate your age in days.

What all computational problems have in common is they have inputs (your birthday and the current date) and they have desired outputs (your age in days.)

The problem is defined by the set of possible inputs, and the relationship between those inputs and the desired outputs.

A solution to a problem is a procedure that can take any input in the set of possible inputs and produce as output the desired output we want.

The First Rule

The first rule in our Pythonista's Guide To All Problems in the Galaxy, is: 1. What are the inputs?

There is one rule before the first rule, the zeroth rule: 0. Don't Panic! If you panic, you're never going to solve the problem.

1. What are the inputs? In this case its very clearly stated: "your birthday and the current date."

What is the set of inputs we want this to work for?

Inputs: two dates (that gives us a set of all possible inputs)

What is the set of valid inputs?

"Assume that the birthday and current date are correct dates ( and no time travel.)" The second date also needs to be after the first one. If we're being good defensive programmers, we should check if this is true.

We're also going to require that the dates are valid dates in the Gregorian calendar, which started on Oct 15, 1582.

Assumptions like this make life easier for programmers. They make it easier to write our code since it has to work for fewer possible inputs.

For most real world problems, its up to you to figure out how to encode the inputs. This is one of the most important decisions you can make in solving a problem.

In this case, the template for the problem indicates how the inputs are represented. There are six parameters: def daysBetweenDates(year1, month1, day1, year2 month2, day2):

What Are the Outputs

Given your birthday and the current date, calculate your age in days.

The problem doesn't specify explicitly what we want the output to be. The best answer is "Return a number giving the number of days between the first date and second date." If we return a number, we can do more things with it later.

Now we have the second step in Pythonista's Guide To All Problems in the Galaxy: 2. What are the outputs?

All we have left to do is the third step: Solve the problem. That is the hard part. Without steps 1 & 2, it is even more difficult.

Next Step

Now we have our Problem Statement: Define a procedure, daysBetweenDates, that takes two dates as inputs and returns the number of days between the first date and second date. Each date is passed in as three numbers giving a valid year, month, and day in the Gregorian calendar. The second date must not be before the first.

We know what the inputs are and we know what the outputs are.

What we want to do is make sure we understand the relationship between the two by working out some examples. Take several dates and work out the number of days in between by hand.

Take the Next Step

We've done three main steps toward solving the problem:

Understand the inputs.

Understand the outputs.

Understand the relationship between the two using some examples.

At this point, you might be ready to start coding, but if the problem is a challenging one you're probably not.

The next thing you should do is consider systematically how a human solves the problem. If you don't understand how to solve the problem yourself in a systematic way, you're not ready to start writing code.

Algorithm Pseudocode

Lets try to write down an algorithm that systematizes how we solved the problem.

We're going to write this as pseudocode, that means we're not worrying about the details of how to write exactly correct Python, we're trying to get they idea down and see if it makes sense:

month1 += 1

We haven't yet accounted for years:

days += days in year1

Should We Implement It

Should we implement our algorithm? The best answer is no. The pseudocode is already fairly complex, and it doesn't handle lots of cases. It doesn't handle: Input dates in same month, month2 before month1 (which can easily happen with a different year,) or Counting days in leap years. We could try to fix the code to try to handle these cases, but our goal as programmers should be to try to find a simple solution first. Something this complicated is very difficult to get correct. Once you start handling lots of special cases, its really hard to get them all right.

Lets start by thinking of a simpler way. Often the way humans solve a problem is not the simplest way, because humans are lazy, we try to find shortcuts instead of doing things in a real simple, mechanical way. On the other hand, computers are not lazy, doing things in a simple mechanical way is great for computers, makes the program easier to write, and makes it more likely for the program to be correct.

Different Approach

Lets think of a really simple way we might solve this problem: Start on the day of the first date, and count the days. Go one day at a time. Keep doing this until we get to our target date.

Pseudocode for a simple mechanical algorithm:

while date1 is before date2:

date1 = advance to next day

days += 1

Simple Mechanical Algorithm

while date1 is before date2:

date1 = advance to next day

days += 1

This seems really brain dead simple: We should do a sanity check to see how long it would take a computer to count 100 years.

We are counting roughly 100 * 365 = 36,500 days

A processor can do about one billion instructions per second. ( One nanosecond per instruction)

Each day will take about 1000 instructions.

36.5 million instructions total, 0.036 seconds.

Don't Optimize Prematurely

Maybe 0.036 seconds is good enough. We need to look at how we're actually going to use this procedure. If we're going to use it for a human to tell how many days old they are in an interactive way, the fact that they have to wait 3 hundredths of a second doesn't matter. Thats plenty fast enough, getting that down to less than a hundredth of a second is not worth the extra effort. If we're using it in a service thats doing this lots of times for lots and lots of different dates, then maybe its not fast enough, and its worth putting time and effort into doing something better. We shouldn't put that time and effort in until we know it should be faster.

Put this in Pythonista's Guide To All Problems in the Galaxy: Don't optimize prematurely!

def daysBetweenDates(year1, month1, day1, year2, month2, day2):
	"""Returns the number of days between year1/month1/day1 and
	year2/month2/day2. Assumes inputs are valid dates in Gregorian calendar."""
	assert not dateIsBefore(year2, month2, day2, year1, month1, day1)
	days = 0
	while dateIsBefore(year1, month1, day1, year2, month2, day2):
		year1, month1, day1 = nextDay(year1, month1, day1)
		days += 1
	return days
